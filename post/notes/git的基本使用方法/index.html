Git的基本使用方法

安装

下载官方网站最新版本：https://git-scm.com/downloads， 支持Windows，Linux/Unix，macOS系统。

Git配置

常用git配置

  1. 配置命令别名

可以给常用命令设置别名，提高使用效率。

别名的配置也使用config命令，比如给git status设置别名st：

1

git config --global alias.st status

修改log显示效果：

1

git config --global alias.lg "log --color --graph --pretty=format:'%C(bold red)%h%C(reset) - %C(bold green)(%cr)%C(bold blue)<%an>%C(reset) -%C(bold yellow)%d%C(reset) %s' --abbrev-commit"

  1. 修改句尾换行设置 不同操作系统句尾换行设置不同，windows为CRLF（回车+换行，即\r\n），linux为LF（换行）， macOS为CR（回车）。当文件在不同操作系统之间传递时可能引起乱码，windows系统需要修改autocrlf设置为true。

    1

    git config --global core.autocrlf true

  2. 配置默认文本编辑器

默认编辑器为vim，可更改为自己熟悉的编辑器。

1

git config [--system|global] core.editor emacs
  1. 配置差异分析工具 默认工具为git diff，可修改为其他工具：kdiff3、tkdiff、meld、xxdiff、emerge、vimdiff、gvimdiff、ecmerge、opendiff等等。

例如要改用vimdiff，可以这样设置：

1

git config [--system|global] merge.tool vimdiff

git配置文件

修改配置文件内容可以直接修改git环境变量。

git有三个级别的配置文件，可以使用git config工具来配置或读取相应的环境变量。

  1. 版本库级配置文件
  2. 位置：gitrepos/.git/config
  3. 配置仅对当前版本库有效

  4. 使用git config --local进行修改

  5. 全局配置文件

  6. 位置：C:/Users/xxx/.gitconfig （windows）

  7. 配置仅对当前用户有效

  8. 使用git config --global进行修改 0

  9. 系统级配置文件

  10. 位置：Windows系统为C:/ProgramData/Git，mac和Linux为/etc/gitconfig

  11. 配置仅对当前用户有效

  12. 使用git config --global进行修改
